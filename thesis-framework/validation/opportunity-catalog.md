## The Infrastructure Stack

These products aren't independent — they form layers:

```
LAYER 4: DATA PRODUCTS (sell to institutions)
├── Credit scoring data (KasaBook + AjoTrust + RideBooks → VendorScore)
├── Price databases (BuildTrack + HarvestPrice → banks, government)
├── Port intelligence (PortPulse → NPA, trade finance)
├── Skills credentials (SkillProof + MasterBook → employers, lenders)
└── Insurance risk data (InsureMatch → reinsurers)

LAYER 3: FINANCIAL PRODUCTS (built on Layer 2 data)
├── InsureMatch (insurance enablement)
├── VendorScore (credit scoring)
├── SupplyLink (invoice factoring)
├── Fractional CP platform
└── Apprenticeship settlement financing

LAYER 2: BUSINESS TOOLS (generate Layer 1 data)
├── PortPulse (port tracking)
├── BuildTrack (construction costs)
├── ClearFast (customs documentation)
├── StockClear (surplus inventory)
├── PharmaDrop (pharmacy supply chain)
└── GasPool (bulk LPG purchasing)

LAYER 1: DATA COLLECTION (invisible infrastructure)
├── KasaBook (informal debt/sales tracking)
├── AjoTrust (savings circle records)
├── RideBooks (driver earnings)
├── MarketPass (stall ownership)
├── SkillProof (artisan verification)
├── MasterBook (apprenticeship documentation)
├── HarvestPrice (commodity prices)
└── Checkpoint tracker, RentRecord, etc.

FOUNDATION: USSD BUSINESS TOOLKIT PLATFORM
└── Session management, user accounts, data storage, SMS delivery
    └── Every Layer 1 product runs on this
```

Build bottom-up: Foundation → Layer 1 (side projects, free) → Layer 2 (revenue products) → Layer 3 (financial products) → Layer 4 (data monetization).

But FUND top-down: Layer 2 products (PortPulse, BuildTrack, ClearFast) generate revenue that funds Layer 1 development (KasaBook, AjoTrust, RideBooks) which creates the data that makes Layer 3 and 4 possible.
